# Update Docs

This skill guides you through updating documentation to reflect code changes.

## Instructions

1. Identify the scope of code changes (new features, bug fixes, refactoring).
2. Locate the relevant documentation files (e.g., `README.md`, `/docs/*.md`).
3. Update the documentation to accurately describe the new behavior, API changes, or configuration options.
4. Ensure examples and tutorials are up-to-date.
5. verify that the documentation is clear, concise, and follows the project's style guide.
